About This Tool
This demo is part of a master's thesis on popularity bias mitigation in POI recommender systems. We chose three users from the Gowalla dataset around Stockhom and visualize the places they visited in the past & the recommendations they received from different algorithms.
User Groups
Users are divided into three groups based on check-in popularity:
g1 - High popularity user
g2 - Mixed popularity user
g3 - Low popularity user
Training Data
Locations that a user has visited in the past, used to calculate future recommendations.
Recommendation Algorithms
Baseline: Standard model (BPR). Recommends many popular places

UPD: Balances recommendations based on average visited location popularity.
Context-Aware POI Recommender: Uses LORE algorithm to factor in time, geography, and visit sequences. Recommends many niche places.
